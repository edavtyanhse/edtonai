# User Story Map & Release Plan

## 🎯 Цели проекта
**Проблема**: Кандидаты тратят часы на адаптацию резюме под каждую вакансию, но часто не могут пройти ATS-фильтры из-за отсутствия ключевых слов или неправильной структуры.
**Решение**: Сервис, который за 30 секунд анализирует вакансию и адаптирует резюме, повышая шансы на интервью.

---

## ✅ Реализованный функционал (MVP + MUP Elements)

Мы реализовали **100% плана MVP** и значительную часть **MUP**:

### MVP (База)
- [x] Авторизация (Supabase Auth).
- [x] Парсинг резюме и вакансий (Groq/Llama 3).
- [x] Анализ совпадения (Match Score).
- [x] Генерация улучшений для Summary.

### MUP (Сверх плана, для оценки 9-10)
- [x] **Генерация Cover Letter**: полноценное сопроводительное письмо.
- [x] **История версий**: возможность вернуться к любому этапу.
- [x] **Hybrid AI**: Использование DeepSeek V3 для сложной логики (Reasoning) и Groq для скорости.
- [x] **Умный парсинг ссылок**: Поддержка HH.ru, LinkedIn, Glassdoor.
- [x] **Сбор обратной связи**: Встроенный механизм Feedback Loop.

## 📊 Сценарии использования (Key Scenarios)

### Сценарий 1: "Быстрая адаптация" (MVP)
1. Пользователь копирует текст своего резюме.
2. Вставляет ссылку на вакансию.
3. Система показывает Score: 45%.
4. Пользователь нажимает "Улучшить".
5. Система переписывает Summary, добавляя ключевые слова.
6. Новый Score: 85%.

### Сценарий 2: "Глубокая проработка" (MUP)
1. Пользователь загружает PDF.
2. Система находит "Missing Skills": *Docker, FastAPI*.
3. Пользователь выбирает галочками: "Добавить навык Docker в опыт работы".
4. Система генерирует новый bullet point: *"Containerized applications using Docker..."*.
5. Пользователь генерирует Cover Letter.
6. Скачивает готовый пакет документов.
