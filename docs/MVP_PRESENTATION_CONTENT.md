# Структура Презентации (MVP Defense)

Этот документ содержит готовый контент для создания презентации (PDF) на защиту.

---

## Слайд 1: Титульный
- **Название проекта**: EdTon.ai
- **Тема**: Умный сервис адаптации резюме под вакансии.
- **Команда**: [Ваши Имена]
- **Этап**: MVP (Minimum Viable Product).

---

## Слайд 2: Проблема и Решение
**Проблема:**
- Рекрутеры тратят 6 секунд на резюме.
- ATS (роботы) отклоняют 75% резюме из-за отсутствия ключевых слов.
- Кандидаты тратят часы на ручную адаптацию под каждую вакансию.

**Решение:**
- EdTon.ai: Адаптация резюме за 3 клика.
- AI анализирует вакансию, находит гэпы и сам переписывает текст, повышая шансы на прохождение фильтров.

## Слайд 3: User Story Map & MVP Scope
*(Вставьте скриншот таблицы из `docs/USER_STORY_MAP.md`)*

**Ключевой функционал MVP:**
1. Парсинг (Текст/URL).
2. Gap Analysis (Сравнение навыков).
3. AI-генерация улучшений.
4. История изменений.

**Сверх плана (MUP Elements):**
- Генерация Cover Letter.
- Гибридная AI-архитектура (DeepSeek + Groq).
- Сбор обратной связи.

---

## Слайд 4: Архитектура (High-Level)
*(Вставьте C4 Context диаграмму из `docs/ARCHITECTURE.md`)*

**Технологии:**
- **Frontend**: React, Vite, Tailwind (SPA).
- **Backend**: FastAPI, Python 3.11 (Async).
- **AI**: DeepSeek V3 (Reasoning) + Groq (Fast Inference).
- **DB/Auth**: Supabase (PostgreSQL).

---

## Слайд 5: Демонстрация (Сценарий)
*(Скриншоты интерфейса)*
1. **Wizard Step 1**: Ввод резюме.
2. **Wizard Step 2**: Вставка ссылки на вакансию.
3. **Analysis Step**: График совпадения (Score) и список недостающих навыков.
4. **Improvement**: Выбор улучшений галочками -> Результат "Было/Стало".
5. **Cover Letter**: Готовое письмо.
